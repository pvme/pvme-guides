# PvME Style Guide
Consistency makes guides easier to read and maintain. This guide covers formatting, structure, and PvME-specific conventions.

.
## __Sections & Templates__
.tag:sections
ÔČą We have various types of guides, each of which has a unique structure and sections within
ÔČą Most templates you will require can be found on the [Guide Editor](<https://pvme.io/guide-editor/>):
.img:https://img.pvme.io/images/By04eu9tkH.png

.
## __Language & Formatting__
.tag:language
ÔČą Use UK English (e.g. armour, not armour; defence, not defense)
ÔČą Date format: 22nd July 2013
ÔČą Do not end lines with blank space (this is referred to as "trailing whitespace")

.
### __Styling & Punctuation__
.tag:styling
ÔČą End full sentences with full stops.
    ÔÇó Exceptions: bullet points or if a paragraph ends in an emoji ­čÜÇ

ÔČą Text styles:
    ÔÇó `**Bold**` ÔÇô Key items or emphases
    ÔÇó `*Italic*` ÔÇô Inline notes or footnotes
    ÔÇó `__Underline__` ÔÇô Optional emphasis

.
### __Emojis__
.tag:emojis
ÔČą **Spacing**: Emojis require a space before **and** after:
```
ÔŁî Detonate<:deto:123> is a spell.
ÔŁî Spellbook Swap <:SBS:543875425055670275>is a spell.
Ôťů Spellbook Swap <:SBS:543875425055670275> is a spell.
```

ÔČą There are a few **exceptions** where spaces are not needed:
    ÔÇó Letter prefixes (`r`, `s`, `_`)
    ÔÇó Punctuation/backslash/backtick prefixes
    ÔÇó Trailing punctuation or underscore
    ÔÇó Inside backticks (embeds)
    ÔÇó Invisible characters
    ÔÇó Allowed examples:
```
r<:emoji:1>  <:emoji:7>.  "ÔÇŽ<:emoji:4>ÔÇŽ"  <:emoji:7>!  `ÔÇŽ<:emoji:11>ÔÇŽ`  <emoji:73>\n\u00a0
```

.
### __Notes__
.tag:notes
ÔČą Use sparingly to highlight key info
ÔČą Put notes *below* their related content
ÔČą For a single note, show it in-line and italic:
```
*Note: This is a note.*
```
ÔČą For multiple notes, show as bullets and italic:
```
*Notes:*
ÔČą *This is a note*
ÔČą *This is another note*
```

.
### __Headings__
.tag:headings
ÔČą Headings represent sections and sub-sections

ÔČą Always put a `.` on the line before any heading (ignore the backslash `\` in the below example)
```
<End of previous section>

\.
## __Section 2__
This is the new section
```

.
ÔČą Use `#` and underline the title
ÔČą Use `##` / `###` and underline level 1 and 2 headings
ÔČą Level 3 headings:
    ÔÇó Discord doesn't allow `####` for level 3 headings
    ÔÇó Therefore remove the underline to distinguish
.
```# __Title__

## __Section 1__
### __Sub-section 1.1__
### Sub-sub-section 1.1.1

## __Section 2__

## __Section 3__
### __Sub-section 3.1__```

ÔČą Headings should be capitalised following the Oxford style:

### __Lists__
.tag:lists
ÔČą Bulleted lists use different characters depending on the level
    ÔÇó Level 2 bullets should be indented 4 spaces
        ÔČę Level 3 bullets should be indented 8 spaces

1) Numbered Lists follow the same rules with indentation
    a) Level 2 should use letters
        ÔČę Level 3 should be bullets

ÔČą Where visually dense, you may add line breaks between list items (like above this bullet)

.
### __Links and Embedded Content__
.tag:embeds
.
ÔČą Only use content from `https://runescape.wiki/`, `https://www.youtube.com/` or `https://img.pvme.io/` (via <#771436827214086154>)
ÔČą Add angled brackets `<>` around links to prevent them showing inline
    ÔÇó For example, `[Greater Fury](<https://runescape.wiki/w/Greater_Fury>)`

.
### __Useful Markdown & GitHub Syntax__
.tag:markdown
ÔČą `` `code` `` for inline code
ÔČą `.img:<url>` for images
ÔČą `.tag:sectionName` for ToC section targets
    ÔÇó `$linkmsg_sectionName$` for linking to that section

.
## __Filenames__
.tag:filenames
ÔČą Filenames should follow these naming patterns:
    ÔÇó Basic guides: `{bossName}-basic.txt`
    ÔÇó Combat Achievements: `{bossName}-ca.txt`
    ÔÇó Commands: `{commandName}.txt`
    ÔÇó DPM Advice: `dpm-advice-{style}.txt`
    ÔÇó One-tick guides: `{bossName}-one-tick.txt`
    ÔÇó Full Boss guides: `{bossName}-{difficulty}-{style}.txt`
        ÔČę `{difficulty}` is optional; either `hm` or `nm`
        ÔČę `{style}` is optional; either `ranged`, `magic`, `melee`, `necromancy`, `hybrid`
    ÔÇó Slayer: `{monsterName}.txt`

ÔČą If not listed above, hyphenate the title:
    ÔÇó e.g., `interface-guide.txt`

ÔČą These naming patterns should also apply if the file is inside the boss folder, e.g. `/sanctum/sanctum-hm-ranged.txt`

.
## __Presets__
.tag:presets
.
ÔČą Create and link to presets in our Preset Maker tool (<#1250465587813945440>)
ÔČą Multiple presets may be created for different situations
ÔČą Links to presets should be in the following pattern:
    ÔÇó `{Combat style} - {Basic/Advanced/Tier}`
        ÔČę e.g., [Necromancy - Basic](#) / [Necromancy - T95/100](#)
    ÔÇó Only the last link should be shown - the rest should be hidden by wrapping the link in `<>`:
        ÔČę e.g., <https://presets.pvme.io?id=presetID>
ÔČą Avoid dyed gear or overpriced items which do not feature in <#1020045708746833930>
ÔČą Be consistent with:
    ÔÇó Rune pouch colours (<#1025569571710718093>)
    ÔÇó EoF emojis (<#1020550560551813131>)

### __Using Notes in Presets__
ÔČą Notes relating to specific preset gear/items should be added to the Preset Maker
    ÔÇó Do not add notes to the guide, for example below the Preset image

ÔČą Add a note for any non-standard gear or niche items
ÔČą Add a note on any gear where perks differ from standard ones listed in <#553632787639435284>

ÔČą For AFK guides, notes may be added to the `Mandatory AFK Requirements` embed for:
    ÔÇó Required ability unlocks (including <:necromancy:1148995625896120460> Talent Tree)
    ÔÇó Mandatory gear/items differing from those listed in <#1251377307315077151>

.
## __Rotations__
.tag:rotations
.
ÔČą Use [full emoji codes](<https://raw.githubusercontent.com/pvme/pvme-settings/master/emojis/emojis.json>), e.g., `<:emoji:ID>`

ÔČą Use `+` to combine same-tick actions, e.g., <:vulnbomb:655341074235129858> + <:deathskulls:1159434663903899728>

ÔČą Use `Ôćĺ` to separate ticks and GCDs, e.g., <:rapid:535541270521708566> Ôćĺ <:snap:535534127131394088>

ÔČą Use `()` for:
    ÔÇó Target cycle (tc)
    ÔÇó Auto attacks (auto)
    ÔÇó Waiting (2t) - ticks are from previous action
    ÔÇó Channel durations (3 hit)
    ÔÇó Weapon types (DW) / (2H)

ÔČą Include non-ability actions, e.g., <:vulnbomb:655341074235129858> <:adrenrenewal:736298121704767538> <:undeadslayer:641339922019516416> <:dragonslayer:641339921814126594> <:demonslayer:641339921675845633> <:nip:537336877900890135>

ÔČą Use `s` and `r` to represent stall & release, e.g.:
    ÔÇó `s<ability> Ôćĺ r<ability> + <nextability>`

ÔČą Use the following example pattern for Detonate:
    ÔÇó <:sonicwave:1087373972443775056> Ôćĺ <:deto:535533833358016512> (3t) Ôćĺ <:bloodbarrage:537338981747261446> + <:deto:535533833358016512> + <:dbreath:535533833391702017> Ôćĺ etc.

.
ÔČą Use <:spec:537340400273195028> and <:eofspec:1257438999794946099> properly, e.g.:
    ÔÇó Weapon specs: `<:sgb> <:spec>` <:sgb:626466665848242186> <:spec:537340400273195028>
    ÔÇó EOF specs: `<:sgb> <:eofspec>` <:sgb:626466665848242186> <:eofspec:1257438999794946099>

ÔČą Show ammo swaps before and after abilities, e.g.:
    ÔÇó Single ammo switch: <:gricocaroming:867678153966878740> Ôćĺ <:fularrow:971025696958853180> <:sgb:626466665848242186> <:eofspec:1257438999794946099> Ôćĺ etc.
        ÔČę Includes off-style abilities: <:hydrixbakri:550834403136503822> <:gricocaroming:867678153966878740> Ôćĺ etc.
    ÔÇó Multiple ammo switch (like "wensporing"): <:gricocaroming:867678153966878740> Ôćĺ <:wenarrow:971025697046925362> <:dbow:643848618553507843> <:eofspec:1257438999794946099> <:deathsporearrows:900758234527301642> Ôćĺ etc.

ÔČą 0-tick abilities should follow this pattern:
    ÔÇó <:roarofawakening:1266417538045317121> <:spec:537340400273195028> (0tick)
    ÔÇó Add a note pointing to the `!0tick` command

.
